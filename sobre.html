Sobre o Projeto

Este projeto foi desenvolvido como atividade prática da disciplina, com o objetivo de aplicar estratégias de branching utilizando Git e GitHub.

Estratégia Escolhida

O grupo utilizou a estratégia GitHub Flow, muito usada em equipes ágeis para desenvolvimento colaborativo.

O que é GitHub Flow?

GitHub Flow é uma estratégia de organização utilizada no desenvolvimento de projetos com Git e GitHub.

Esse modelo é baseado no uso de branches, permitindo que diferentes pessoas trabalhem no mesmo projeto sem alterar diretamente a branch principal.

Qual a funcionalidade?

Sua principal funcionalidade é permitir que várias pessoas trabalhem ao mesmo tempo no mesmo projeto de forma organizada e segura.

Cada integrante cria uma branch separada para desenvolver sua funcionalidade, evitando conflitos no código principal.

Para que serve?

O GitHub Flow serve para facilitar o trabalho em equipe, organizar alterações no projeto e melhorar o controle de versões.

Além disso, permite revisar mudanças através de Pull Requests, realizar Code Review e integrar novas funcionalidades sem comprometer a branch main.

Como Funcionou no Projeto

  • Cada integrante criou sua própria branch
  • As alterações foram feitas separadamente
  • Os arquivos foram enviados através de Pull Requests
  • Os colegas realizaram Code Review
  • Após aprovação, foi realizado o merge para a branch main
