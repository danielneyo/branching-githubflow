## Objetivo do Projeto

Este projeto tem como objetivo praticar o uso do GitHub Flow
utilizando branches, commits, push, pull requests e merge.

A atividade busca desenvolver a colaboração em equipe,
organização do código e utilização de boas práticas
de versionamento com Git e GitHub.

---

## Sobre o GitHub Flow

O GitHub Flow é uma metodologia simples de controle
de versões utilizada para organizar o desenvolvimento
de projetos em equipe.

Seu principal objetivo é permitir que vários
desenvolvedores trabalhem ao mesmo tempo sem
comprometer a estabilidade do projeto principal.

---

### Etapas do GitHub Flow

### 1. Criar uma Branch
Cada funcionalidade deve ser desenvolvida em uma branch separada.

### 2. Fazer Alterações
O desenvolvedor realiza modificações no projeto sem afetar a branch main.

### 3. Realizar Commits
As alterações são salvas através de commits com mensagens descritivas.

### 4. Fazer Push
Os commits são enviados para o repositório remoto no GitHub.

### 5. Abrir Pull Request
É criada uma solicitação para revisar e integrar o código.

### 6. Revisar Código
Os integrantes analisam o código e sugerem melhorias.

### 7. Fazer Merge
Após aprovação, as alterações são integradas na branch principal.
